📦 MÓDULO 1: Instalación y Configuración

Todo lo que necesitás para tener tu CRM funcionando. Un solo comando instala todo.

1.1 Descargar Termux Humano

Termux es la app que convierte tu Android en una computadora. Es gratis y segura.

  1. Descargá F-Droid desde f-droid.org (NO de Play Store)
  2. Abuí F-Droid y buscá "Termux"
  3. Instalalo y abrilo
⚠️ Importante: La versión de Play Store está desactualizada. Usá SIEMPRE F-Droid.
1.2 Preparar la carpeta Humano

Cuando descomprimís el .zip, la carpeta queda en Descargas. Necesitás moverla:

# Dar permisos
termux-setup-storage

# Mover la carpeta a la raíz
cp -r ~/storage/downloads/curso-crm ~/

# Entrar a la carpeta
cd ~/curso-crm
1.3 Instalación automática Agente

Ejecutá el instalador. Instala todo automáticamente:

chmod +x install.sh
./install.sh

El instalador hace 9 pasos:

  1. Mueve la carpeta a la raíz
  2. Da permisos de almacenamiento
  3. Actualiza Termux
  4. Instala dependencias (Python, Node.js, Git)
  5. Instala librerías Python
  6. Instala Hermes Agent (tu asistente IA)
  7. Configura tu proveedor de IA ← Paso clave
  8. Configura Zira (personalidad del bot)
  9. Configura WhatsApp (con QR)
💡 Recomendación: Elegí OpenCode Zen (opción 1) — es gratis, tiene un buen modelo, y el uso es generoso.
1.4 Iniciar a Zira Agente

Después de la instalación, Zira está lista. Solo escribí:

hermes

Zira se despierta y te guía en el resto de la configuración. Ella puede:

  • Explicar cada paso del CRM
  • Ayudarte a configurar Google Cloud
  • Configurar WhatsApp e Instagram
  • Resolver dudas en lenguaje simple
📊 MÓDULO 2: Google Cloud — Base de Datos del CRM

Google Sheets como base de datos central y Calendar para reservas. La parte más burocrática — seguí cada paso al pie de la letra.

2.1 Crear proyecto en Google Cloud Humano Google

Esto es lo que la IA no puede hacer por vos — requiere login y aceptación de términos:

  1. Entrá a console.cloud.google.com
  2. Arriba a la izquierda, hacé clic en "Seleccionar proyecto" → "Nuevo proyecto"
  3. Dale un nombre genérico (ej: "CRM Hotelero") y crealo
  4. En el menú lateral, andá a "APIs y servicios" → "Habilitar APIs y servicios"
  5. Buscá y activá: Google Sheets API y Google Calendar API
⚠️ Importante: Anotá el Número de proyecto que aparece en la pantalla principal. Lo vas a necesitar.
2.2 Crear Cuenta de Servicio Humano Google

La Cuenta de Servicio es como un "usuario robot" de Google que puede leer/escribir en tus hojas sin que tengas que hacer clic en "Permitir" cada vez:

  1. En el menú lateral, andá a "IAM y administración" → "Cuentas de servicio"
  2. Hacé clic en "Crear cuenta de servicio"
  3. Dale un nombre (ej: "agente-crm") y creala
  4. Hacé clic en la cuenta creada → pestaña "Claves" → "Agregar clave" → "Crear clave nueva"
  5. Seleccioná JSON y descargalo
  6. Renombrá el archivo descargado como credentials.json
  7. Colocalo en la raíz del proyecto (junto a .env)
⚠️ Seguridad: Este archivo es como la llave maestra. NUNCA lo subas a GitHub. Ya está en .gitignore.
2.3 Compartir la hoja de cálculo Humano Google

Google Sheets no permite acceder a hojas privadas sin permiso. Debes "compartir" la hoja con la Cuenta de Servicio:

  1. Creá una hoja nueva en Google Sheets
  2. Hacé clic en "Compartir" (arriba a la derecha)
  3. Pegá el correo largo de la Cuenta de Servicio (algo como agente-crm@tu-proyecto.iam.gserviceaccount.com)
  4. Asignale permisos de Editor
  5. Copiá el ID de la hoja de la URL (la parte entre /d/ y /edit)
  6. Pegalo en tu .env como GOOGLE_SPREADSHEET_ID
2.4 Verificar la conexión Agente

El agente ejecuta una prueba para confirmar que puede leer la hoja:

# El agente ejecuta esto internamente
python3 -c "
from crm.google_auth import obtener_cliente
from crm.connectors import SheetsConnector
cliente = obtener_cliente()
sheets = SheetsConnector(cliente)
print('Conexión exitosa con Google Sheets')
"

Si ves "Conexión exitosa", todo está funcionando.

♾️ MÓDULO 3: Meta — WhatsApp e Instagram Business API

El laberinto más complejo. Meta tiene muchas vueltas — pero una vez que lo superás, tenés los canales de venta y captación automatizados.

3.1 Crear App en Meta Developers Humano Meta
  1. Entrá a developers.facebook.com con tu cuenta de Facebook
  2. Hacé clic en "Mis apps" → "Crear app"
  3. Seleccioná tipo "Negocio"
  4. Dale un nombre y crealo
  5. En "Agregar productos", habilitá: WhatsApp e Instagram Graph API
3.2 Configurar WhatsApp Business Humano Meta
  1. En el panel de WhatsApp, andá a "Configuración de la API"
  2. Copiá el Token de acceso temporal (para pruebas)
  3. Copiá el ID del número de teléfono
  4. Pegalo en tu .env
⚠️ Token temporal vs permanente: El token de prueba expira en 24hs. Para producción, creá un Token de Usuario del Sistema Permanente en Business Manager → Configuración → Usuarios del Sistema.
3.3 Verificar número de teléfono Humano Meta

Meta requiere que verifiques que sos dueño del número:

  1. En el panel de WhatsApp, ingresá tu número
  2. Elegí recibir el código por SMS o llamada
  3. Ingresá el código de 6 dígitos

Estado esperado: "Conectado" ✅

3.4 Crear plantilla de mensaje Humano Meta

Para enviar mensajes proactivos (fuera de la ventana de 24h), necesitás una plantilla aprobada por Meta:

  1. En el panel de WhatsApp, andá a "Plantillas de mensaje"
  2. Hacé clic en "Crear plantilla"
  3. Elegí categoría: UTILITY o MARKETING
  4. Escribí el texto con variables: Hola {{1}}! Bienvenido a nuestra posada.
  5. Paso crítico: Agregá un "ejemplo" para cada variable (Meta lo exige)
  6. Enviá a revisión — puede tardar minutos o horas
⚠️ Error común: Si Meta rechaza la plantilla por "Variables sin texto de muestra", es porque olvidaste poner el ejemplo de cada variable.
3.5 Configurar Webhook Humano Agente

El Webhook es la dirección donde Meta envía los mensajes que recibís. Para desarrollo local:

  1. Instalá un túnel (en otra pestaña de Termux):
# Opción 1: localhost.run (gratis, sin cuenta)
ssh -R 80:localhost:3000 nokey@localhost.run

# Opción 2: ngrok (requiere cuenta gratis)
ngrok http 3000
  1. Copiá la URL pública que te dé el túnel
  2. En Meta Developers → WhatsApp → Configurar Webhook
  3. Pegá la URL + /webhook
  4. Ingresá tu META_VERIFY_TOKEN del .env
  5. Hacé clic en "Verificar y guardar"

El agente levanta el servidor automáticamente:

node core-crm/webhookServer.js
3.6 Configurar Instagram Business Agente

Para que Zira Bot pueda responder comentarios y enviar DMs:

  1. Vinculá tu cuenta de Instagram a una Página de Facebook
  2. En Meta Developers, habilitá Instagram Graph API
  3. Generá un token con permisos: instagram_basic, instagram_manage_messages, pages_messaging
  4. Pegalo en .env como INSTAGRAM_ACCESS_TOKEN
🐙 MÓDULO 4: GitHub — Respaldo y Versiones

GitHub es la memoria del proyecto. Si el celular se apaga o se pierde, tu código está seguro en la nube.

4.1 Crear cuenta y token Humano GitHub
  1. Creá una cuenta en github.com (si no tenés)
  2. Andá a Settings → Developer settings → Personal access tokens → Tokens (classic)
  3. Generá un nuevo token con permisos: repo, workflow
  4. Copialo y pegalo en .env como GITHUB_TOKEN
4.2 Crear repositorio Agente

El agente crea el repositorio y sube el código automáticamente:

# El agente ejecuta:
git init
git add .
git commit -m "Initial CRM setup"
git remote add origin https://github.com/TU_USUARIO/tu-repo.git
git push -u origin main
4.3 Deploy automático Agente

El agente crea un workflow de GitHub Actions para deploy automático cada vez que hagas push:

# .github/workflows/deploy.yml
name: Deploy CRM
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy
        run: echo "Deploying CRM..."
